# NYC 311 Service Equity Analytics Pipeline

Analyzing complaint resolution delays across boroughs, agencies, and service 
categories using SQL, Python, PostgreSQL, and Power BI.

## Problem Statement
Are NYC 311 complaints resolved at the same speed across different boroughs, 
agencies, and complaint types? This project investigates service equity and 
operational bottlenecks in NYC's public complaint system using 2023–2025 data.

## Tools Used
- Python 3.11
- Pandas
- PostgreSQL
- SQL
- Power BI
- Git/GitHub
- Jupyter Notebook

## Dataset
NYC 311 Service Requests from 2020 to Present — NYC Open Data  
Filtered to 2023–2025 and six complaint categories:
HEAT/HOT WATER, Noise, Illegal Parking, Street Condition, Rodent, Water System

## Project Structure
- data/ — raw and processed datasets
- src/ — Python ETL scripts
- sql/ — SQL schema, KPI, and analysis queries
- notebooks/ — exploratory analysis
- dashboard/ — Power BI screenshots
- docs/ — methodology, architecture, data dictionary, final insights

## Key Questions
1. Which complaint types take the longest to resolve?
2. Which boroughs have the highest delay rates?
3. Which agencies handle the most delayed complaints?
4. Do delays change by season or month?
5. What recommendations could improve service response?
